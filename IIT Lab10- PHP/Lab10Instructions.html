Lab 10: Databases & SQL

Intro ITWS

Import, modify, and export a database

We will do most of this together in class.

  • I have provided you with an SQL file with the commands to create an actors table and insert some actors. I have also provided you with a PHP program which has a menu with two tabs. One will add/remove/display actors, however there is no table for the actors. The other is to do the same for movies, and while I have given you the table, I have not given you the code for the movies.php program.
  • In this lab, you are going to complete the program so that we can Add/Remove and display both movies and actors.
  • Create a Lab 10 issue and branch to your repos
  • Download the lab: ITWS1100-LAB10.zip from LMS
  • Guidelines:
      • Add a lab10 folder to your /xampp/htdocs/iit/ directory (or its equiv)
      • NOTE: You must use your LAMP servers to complete this lab
      • unzip the lab10 zip file from LMS into your new lab10 folder in your iit directory
      • Make sure you start your XAMPP server and your database server
      • Using phpMyAdmin, (Mac VM), create a new database named "iit", setting the collation to utf8_general_ci
      • In phpMyAdmin with the "iit" database selected, import lab10/iit-lab10start.sql
    2. Using the create table commands in the iit-lab10start.sql file as guide, create a new table named "actors" in which to store character names of the roles played by our actors. Include four fields, an autoincrement primary key, first, last name and a date of birth (for the characters).
      Note: Check the code for the proper naming for the fields.
    3. Using phpMyAdmin, add a few actors that you like to the table
    4. Using the insert statements from the iit-lab10start.sql file as a guide, or using the insert statement from the lecture slides, add at least five new actors - not mine - to the actors table.
    5. Using the format of the dates in the actors table as a guide, query the database for all actors born on or after 1960. (Remember to put the date string in single quotes.) Output your results to a CSV file using the "Export" tab in phpMyAdmin after you've made the query. Export the file as "CSV for MS Excel". Name the CSV file "actors.csv" and save it in your lab10/ folder.
    6. Using the examples we went over in class from iitajax, create a relationship table for movies and actors, and associate some of your actors to their movies.
    7. Using the examples we went over in class, complete the program to list out movies as the example did for actors.
    8. For extra credit, add a panel (tab) to the program to list out the movies and their corresponding actors.
    9. Select "iit" in the left column to select the database. Using the Export tab in phpMyAdmin, export the SQL schema by selecting the SQL radio button in the export box (left column). Save it as iit.sql in your lab10 folder. Check this file to make sure you have the full schema (not, for example, the last table you looked at).
    10. When you are done,
      • commit and push your changes to your repo
      • Zip up your iit directory and rename it to lab10-YourName-yourRCSID.zip
      • submit your zip file to LMS
    11. This lab is due by midnight on date shown in LMS
